## ソフトのインストールとパッケージマネージャ
これからプログラミングやデータの解析で多くのソフトウェアを扱うことになりますが、ソフトウェアを実際に使えるようにすること(インストール)は、大変難しいこともあります。

こうした問題を解決するため、ソフトウェア(ソフトウェアパッケージとも呼ばれます)をインストールし、管理するためにパッケージマネージャと呼ばれるソフトが使われるようになりました。

パッケージマネージャは、非常に便利なツールです。特に`pip`と`conda`がよく使われます。ただ、この２つは似ているのですが動作が異なるため、**どちらか１つを使うように気をつけてください**。

バイオインフォマティクスの世界では、`conda`の方が使われることが多いので、以下は`conda`の説明になります。


## condaのインストール方法

`conda`の中にもいくつか種類があり、最近有償化したりしているので、無償で提供されているminiforge版とよばれる`conda`を紹介します。

```
curl -L -O "https://github.com/conda-forge/miniforge/releases/latest/download/Miniforge3-$(uname)-$(uname -m).sh"
```
もしくは

```
wget "https://github.com/conda-forge/miniforge/releases/latest/download/Miniforge3-$(uname)-$(uname -m).sh"
```

１つめの`curl`で始まる行をターミナルにコピーし、もし失敗した場合は`wget`で試してみてください。どちらでも同じですが、環境によっては失敗するかもしれないので、２つ書いておきます。

ダウンロードが終わったら、以下を実行するとminiforge版の`conda`がインストールされます。

```
bash Miniforge3-$(uname)-$(uname -m).sh
```

一度インストールが終わったら、ターミナルの再起動(終了して、再度起動)を行い以下のコマンドを入力してください。

```
conda
```

condaの使い方が表示されれば、成功です！
まずは、よく使うpythonをインストールしてみましょう。

```
conda install python
```




## (発展) pipとcondaの違い

pip と conda は、Pythonパッケージの管理に使用されるツールですが、それぞれ異なる特徴と用途があります。

pip
- 用途: Pythonパッケージのインストールと管理。
- パッケージソース: Python Package Index (PyPI) からパッケージをインストール。
- 依存関係: Pythonパッケージのみを管理。システムパッケージや他の言語のライブラリは管理しない。
- 環境管理: 仮想環境ツール（virtualenvやvenv）と組み合わせて使用。

conda
- 用途: クロスプラットフォームのパッケージと環境管理。
- パッケージソース: Anacondaリポジトリやconda-forgeからパッケージをインストール。
- 依存関係: Pythonパッケージだけでなく、CやC++のライブラリ、Rパッケージなども管理。
- 環境管理: conda自体が環境管理機能を持ち、異なるバージョンのPythonやパッケージを含む環境を作成可能

バイオインフォマティクスの場合、Python以外で書かれたソフトウェアを使うことが非常に多いため`conda`が使われることが多い印象です。
特に、`bioconda`という分子生物学に特化した`conda`のコミュニティがあり、ここの存在は大きいです。
